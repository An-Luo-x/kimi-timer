# 构建与打包指南

本文档指导如何在开发环境中运行 Kimi Timer，以及如何将其打包为独立的 Windows 可执行文件。

---

## 一、开发环境运行

### 1.1 前提条件

- 已安装 Node.js（推荐 v20+）和 npm
- 已克隆/下载本项目到本地

### 1.2 启动步骤

```bash
# 进入项目目录
cd kimi-timer

# 安装依赖（首次）
npm install

# 启动开发环境
npm start
```

窗口将自动弹出，显示计时器界面。

### 1.3 开发环境特点

- 自动加载当前目录下的 `main.js` 和 `index.html`
- 支持实时修改代码后重启（需关闭窗口后重新 `npm start`）
- 控制台输出可用于调试（运行 `npm start` 的终端会显示日志）

## 二、打包为 Windows 可执行文件

### 2.1 依赖检查

确保 `electron-builder` 已安装：

```bash
npm ls electron-builder
```

若未安装，执行：

```bash
npm install electron-builder --save-dev
```

### 2.2 打包命令

```bash
npm run build
```

该命令调用 `electron-builder --win --x64`，构建 Windows 64 位版本。

### 2.3 打包输出

打包完成后，生成以下文件：

| 输出 | 路径 | 说明 |
|------|------|------|
| **便携版 .exe** | `dist/Kimi Timer 1.0.0.exe` | **推荐使用**，单文件，双击即运行 |
| 解压版目录 | `dist/win-unpacked/Kimi Timer.exe` | 含完整依赖文件，可进一步自定义 |

### 2.4 打包配置（package.json）

```json
{
  "build": {
    "appId": "com.kimi.timer.desktop",
    "productName": "Kimi Timer",
    "directories": {
      "output": "dist"
    },
    "win": {
      "target": {
        "target": "portable",
        "arch": ["x64"]
      }
    },
    "files": [
      "main.js",
      "index.html",
      "package.json",
      "node_modules/**/*"
    ]
  }
}
```

**关键配置说明：**

- `target: "portable"` — 生成单文件 `.exe`，无需安装
- `arch: ["x64"]` — 仅构建 64 位版本，减小体积
- `directories.output: "dist"` — 输出到 `dist/` 目录

## 三、常见问题与处理

### 3.1 问题：Git Bash 下打包失败，提示 `spawn powershell.exe ENOENT`

**原因：** Git Bash 环境的 `PATH` 未包含 `WindowsPowerShell` 目录，electron-builder 依赖 PowerShell 执行依赖收集脚本。

**解决方案：** 手动将 PowerShell 路径加入环境变量后打包：

```bash
# Git Bash 环境
export PATH="$PATH:/c/Windows/System32/WindowsPowerShell/v1.0"
npm run build
```

或使用 Windows CMD：

```cmd
# CMD 环境
cd C:\Users\MissLi\Desktop\kimi-timer
npm run build
```

### 3.2 问题：Electron 下载超时或失败

**原因：** 默认 Electron 镜像在国外，网络不稳定。

**解决方案：** 已配置国内镜像源，`.npmrc` 文件内容：

```
registry=https://registry.npmmirror.com
electron_mirror=https://npmmirror.com/mirrors/electron/
```

若首次安装仍失败，可尝试：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### 3.3 问题：打包生成的 .exe 文件太大

**说明：** 便携版约 87 MB 为正常体积，内含完整的 Chromium 内核和 Node.js 运行时。若需减小体积，可考虑：

- 使用 `asar` 压缩（electron-builder 默认已启用）
- 移除未使用的 `node_modules` 文件（通过 `files` 白名单精确控制）
- 未来使用 `electron-builder` 的 `compression` 配置项

### 3.4 问题：如何更换应用图标

**步骤：**

1. 准备 ICO 格式图标文件（推荐尺寸：256×256 或 512×512）
2. 在项目根目录创建 `build/` 文件夹
3. 将图标命名为 `icon.ico` 放入 `build/` 目录
4. 在 `package.json` 的 `build.win` 中添加：

```json
"win": {
  "icon": "build/icon.ico"
}
```

5. 重新运行 `npm run build`

## 四、高级打包选项

### 4.1 构建 NSIS 安装包

如需生成传统安装程序（`.exe` 安装向导），修改 `package.json`：

```json
"win": {
  "target": {
    "target": "nsis",
    "arch": ["x64"]
  }
}
```

打包后将生成：
- `dist/Kimi Timer 1.0.0.exe` — 安装程序（可自定义安装路径、创建快捷方式）
- `dist/nsis-web/` — 在线安装版（体积更小）

### 4.2 构建多平台版本

```bash
# 同时构建 Windows 和 macOS
npm run build -- --win --mac

# 构建 Linux 版本
npm run build -- --linux
```

> 注意：跨平台构建需对应平台的构建环境（如 macOS 构建需在 macOS 系统或 CI 环境下执行）。

### 4.3 清理打包缓存

```bash
# 删除 dist 目录和打包缓存
rm -rf dist/
npm run build
```
