## 3. 精准计时需求（核心）

| 编号 | 需求描述 | 实现状态 | 实现细节 |
|------|---------|---------|---------|
| AC-01 | 时间戳差值算法 | ✅ 已实现 | 记录 `startTimestamp = Date.now()`，实时计算 `Date.now() - startTimestamp` |
| AC-02 | 正计时差值计算 | ✅ 已实现 | `elapsedAccumulated + (now - startTimestamp)`，暂停时累加已运行时间 |
| AC-03 | 倒计时差值计算 | ✅ 已实现 | `remainingAccumulated - (now - startTimestamp)`，暂停时保存剩余时间 |
| AC-04 | 后台运行保持精准 | ✅ 已实现 | 基于系统时间戳，不依赖渲染帧率，最小化/后台不影响精度 |
| AC-05 | 合理更新频率 | ✅ 已实现 | 100ms 更新间隔，平衡 CPU 占用与显示流畅度 |
| AC-06 | 暂停/恢复支持 | ✅ 已实现 | 暂停时固化已运行/剩余时间，恢复时重新记录时间戳 |

## 4. 提醒功能需求

| 编号 | 需求描述 | 实现状态 | 实现细节 |
|------|---------|---------|---------|
| AL-01 | 倒计时结束触发提醒 | ✅ 已实现 | `remaining <= 0` 时触发 `onTimerFinish()` |
| AL-02 | Web Audio API 蜂鸣声 | ✅ 已实现 | 三段递进式正弦波：880Hz → 1100Hz → 1320Hz，各 1.2s 衰减 |
| AL-03 | 无需外部音频文件 | ✅ 已实现 | 纯代码生成 `AudioContext` + `OscillatorNode` + `GainNode` |
| AL-04 | Windows 系统原生通知 | ✅ 已实现 | 通过 IPC 调用 Electron `Notification`，标题「计时器提醒」，内容「倒计时已结束！」 |
| AL-05 | 界面视觉闪烁提醒 | ✅ 已实现 | CSS `@keyframes screen-flash` 改变背景渐变色，持续 3 秒 |
| AL-06 | 多重提醒叠加 | ✅ 已实现 | 声音 + 通知 + 视觉同时触发，确保用户感知 |
| AL-07 | 提醒可停止 | ✅ 已实现 | 点击「重置」按钮立即清除所有提醒效果（闪烁、文字发光） |
