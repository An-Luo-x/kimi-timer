## 文件职责详解

### `main.js` — 主进程（Main Process）

**核心职责：** 窗口管理、系统级交互、IPC 通信

| 功能模块 | 实现说明 |
|---------|---------|
| `createWindow()` | 创建 `BrowserWindow`，设置 400×500 固定窗口，隐藏菜单栏 |
| `win.setMenu(null)` | 彻底移除顶部菜单栏 |
| `win.setAlwaysOnTop()` | 通过 IPC 接收渲染进程指令，动态切换置顶状态 |
| `Notification` | 触发 Windows 原生系统通知（由 IPC 桥接） |
| `app.whenReady()` | 应用生命周期：就绪时创建窗口，激活时恢复窗口 |
| `app.on('window-all-closed')` | 非 macOS 平台时退出应用 |

**IPC 通道：**

| 通道名 | 方向 | 数据 | 说明 |
|--------|------|------|------|
| `toggle-always-on-top` | 渲染 → 主进程 | `{boolean} isOnTop` | 切换窗口置顶状态 |
| `show-notification` | 渲染 → 主进程 | `{string} title, {string} body` | 触发系统通知 |

**设计决策：** 主进程保持极简，不处理任何计时逻辑，仅作为渲染进程与操作系统之间的桥梁。

### `index.html` — 渲染进程（Renderer Process）

**核心职责：** 用户界面、计时逻辑、状态管理、提醒触发

| 模块 | 文件范围 | 说明 |
|------|---------|------|
| 界面结构 | `<body>` 内 DOM 元素 | 顶部栏、模式切换、计时显示、设置区、控制按钮、状态栏 |
| 样式系统 | `<style>` | 深色渐变主题、Flex 布局、响应式动画、按钮渐变、状态反馈 |
| 计时引擎 | `updateTimer()` 函数 | 时间戳差值算法，核心核心 |
| 模式切换 | `setMode()` | 倒计时/正计时切换，联动 UI 显隐 |
| 控制逻辑 | `startTimer()` / `pauseTimer()` / `resetTimer()` | 状态机管理 |
| 提醒系统 | `playBeepSound()` / `onTimerFinish()` | Web Audio + IPC + CSS 动画 |
| 输入处理 | `clampInput()` + 事件监听 | 防越界、Enter 键快捷启动 |

**计时引擎核心算法（时间戳差值）：**

```javascript
// 正计时
elapsed = elapsedAccumulated + (Date.now() - startTimestamp);

// 倒计时
remaining = remainingAccumulated - (Date.now() - startTimestamp);
```

**优势：** 不依赖 `setInterval` 的累积误差，即使窗口被最小化、后台运行、系统休眠恢复，时间计算基于真实的系统时间戳，绝对精准。

**状态机：**

```
[IDLE] --start--> [RUNNING] --pause--> [PAUSED] --start--> [RUNNING]
  ↑                                               |
  └--------------- reset -------------------------┘
```

## 架构设计原则

1. **主进程极简化**：主进程不做业务逻辑，只负责窗口管理和系统桥接，所有业务逻辑放在渲染进程。
2. **单页无框架**：使用原生 HTML/CSS/JS，不引入 React/Vue 等框架，减少依赖体积和复杂度。
3. **精准计时优先**：采用时间戳差值而非 `setInterval` 计数，确保计时精度不受系统调度影响。
4. **零外部资源依赖**：蜂鸣声通过 Web Audio API 代码生成，不依赖外部音频文件，减少打包体积和文件丢失风险。
5. **IPC 单向通信**：渲染进程 → 主进程单向调用，主进程不主动推送数据，降低耦合。

## 数据流

```
用户操作（点击/输入）
    ↓
渲染进程 DOM 事件处理
    ↓
计时引擎状态更新（start/pause/reset）
    ↓
updateTimer() 计算时间戳差值 → 更新 display
    ↓
（触发提醒时）
    ├─→ Web Audio API 播放蜂鸣声
    ├─→ IPC → 主进程 → Windows Notification
    └─→ CSS 动画触发界面闪烁
```

## 安全考量

- `nodeIntegration: true` + `contextIsolation: false`：V1.0 为简化开发启用，后续版本建议启用 `contextIsolation` 并预加载脚本。
- 无网络请求：应用纯本地运行，无外部 API 调用，无数据收集。
