# 开发环境搭建指南

本文档面向新接手本项目的开发者，指导从零开始搭建完整的开发环境，直到成功运行 `npm start`。

---

## 前置要求

| 软件 | 最低版本 | 下载地址 |
|------|---------|---------|
| Node.js | 20.0.0 | https://nodejs.org/ |
| npm | 10.0.0 | 随 Node.js 安装 |
| Git（可选） | 2.40.0 | https://git-scm.com/ |

> 验证安装：在终端执行 `node -v` 和 `npm -v`，应显示版本号。

---

## 步骤一：获取项目代码

### 方式 A：通过 Git 克隆（推荐）

```bash
git clone <项目仓库地址>
cd kimi-timer
```

### 方式 B：直接下载 ZIP

1. 下载项目 ZIP 压缩包
2. 解压到本地目录（如 `C:\Users\<用户名>\Desktop\kimi-timer`）
3. 进入解压后的目录

---

## 步骤二：配置国内镜像源（中国大陆用户）

为避免 Electron 下载超时，已内置 `.npmrc` 配置。若你所在网络正常，可跳过此步骤。

确认 `.npmrc` 文件存在且内容正确：

```
registry=https://registry.npmmirror.com
electron_mirror=https://npmmirror.com/mirrors/electron/
```

若无此文件，手动创建：

```bash
# 在项目根目录执行
npm config set registry https://registry.npmmirror.com
```

---

## 步骤三：安装依赖

在项目根目录执行：

```bash
npm install
```

该命令将：
- 读取 `package.json` 中的依赖列表
- 下载 `electron` 和 `electron-builder` 到 `node_modules/`
- 通过国内镜像下载 Electron 平台二进制文件（首次约 1-2 分钟）

**安装成功标志：** 终端显示 `added XX packages`，无报错。

**常见问题：**

| 问题 | 解决方案 |
|------|---------|
| 下载超时/失败 | 检查网络连接，确认 `.npmrc` 镜像配置正确 |
| `npm: command not found` | 确认 Node.js 已正确安装，并加入系统 PATH |
| 权限不足（Linux/Mac） | 使用 `sudo npm install` 或修改目录权限 |

---

## 步骤四：验证开发环境

执行开发启动命令：

```bash
npm start
```

**预期结果：**
- 终端显示 `> electron .`
- 弹出 400×500 的计时器窗口，显示 "Hello World" 或计时器界面
- 窗口标题为「桌面计时器」或「Kimi Timer」

**若窗口未弹出：**

1. 检查终端是否有错误信息
2. 确认 `main.js` 和 `index.html` 文件存在于项目根目录
3. 尝试重新安装依赖：`rm -rf node_modules && npm install`（或手动删除后重装）

---

## 步骤五：验证打包环境（可选）

确保打包工具可用：

```bash
npm run build
```

**预期结果：**
- 终端显示 `electron-builder` 构建日志
- 最终输出 `dist/Kimi Timer 1.0.0.exe`

**Git Bash 用户注意：** 若报错 `spawn powershell.exe ENOENT`，请使用 CMD 环境执行：

```cmd
cd C:\Users\<用户名>\Desktop\kimi-timer
npm run build
```

```bash
# 1. 进入项目
cd kimi-timer

# 2. 修改代码（main.js / index.html）

```

## 常见问题速查

**Q1：为什么用 `npm.cmd` 而不是 `npm`？**  
在 Git Bash 环境下，npm 需通过 `npm.cmd` 调用。在 CMD/PowerShell 中直接使用 `npm` 即可。

**Q2：Electron 窗口白屏/空白？**  
检查 `index.html` 文件是否存在且路径正确。`main.js` 中通过 `win.loadFile('index.html')` 加载，需确保文件在同级目录。

**Q3：如何调试主进程？**  
在 `main.js` 的 `createWindow()` 中添加：

```javascript
win.webContents.openDevTools();
```

启动后将自动打开 Chrome DevTools。

**Q4：修改代码后为什么不生效？**  
Electron 不会自动热重载，需关闭窗口后重新执行 `npm start`。
